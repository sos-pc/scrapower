## 1.3 Surface de sécurité

| Endpoint | Méthode | Auth | Impact |
|----------|---------|------|--------|
| `/transcribe` | POST | API key | Crée tâche whisper |
| `/results/{id}` | GET | API key | Lit résultat |
| `/tasks` | POST | API key | Crée tâche générique |
| `/tasks/{id}` | GET | API key | Lit statut |
| `/blobs` | PUT | API key ou assignment_token | Stocke blob |
| `/blobs/{hash}` | GET | Public | Lit blob |
| `/worker/pull` | POST | API key (obligatoire depuis B9) | Assigne tâche |
| `/worker/submit` | POST | assignment_token | Complete tâche |
| `/worker/heartbeat` | POST | assignment_token | Maintient assignation |
| `/worker/ws` | WS | API key (upgrade) | Mode A |
| `/stats` | GET | API key | Infra status |
| `/health` | GET | Public | Health check |

**Anomalies** :
- `/blobs/{hash}` public : un attaquant avec un hash valide peut télécharger n'importe quel blob. Contournable (les hash sont SHA256 → préimage-resistant).
- `/worker/submit` vérifie `assignment_token` mais pas l'API key : cohérent (le token est l'auth du worker pour cette tâche).

## 1.5 Points de défaillance

| Composant | Panne | Conséquence | Récupération |
|-----------|-------|-------------|--------------|
| Coordinator | Crash | Tâches ASSIGNED orphelines | `requeue_stale` au redémarrage (90s) |
| Worker HF | Crash | Perd la tâche en cours | `_touch_starting` expire (90s) → reassign |
| Worker Kaggle | Kernel killed | Perd la tâche | `cleanup_stale` détecte → reassign |
| Worker Modal | Sandbox expired | Perd la tâche | `cleanup_stale` → reassign |
| Caddy | Crash | Workers externes déconnectés | Docker restart auto |
| VPN WireGuard | Down | Workers ne peuvent pas DL YouTube | Fallback coordinator DL (exit_code=2) |
| DB SQLite | Corruption | Toutes les tâches perdues | Backup cron quotidien |
| HF API | Down | Pas de nouveau déploiement | Retry au prochain tick |
| Kaggle API | Rate limit | Pas de nouveau kernel | Retry après cooldown |
